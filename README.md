## 자바의 예외처리
### 예외 : 실행중 오류나 악영향을 미치는 상황
- 자바에서는 실행 중 발생하는 에러를 예외로 처리

### 실행중 예외가 발생하면 : 응용 프로그램이 예외 처리 하도록 호출
- 응용프로그램이 예외를 처리하지 않으면 프로그램 강제 종료

## 예외 처리문 try-catch-finaily문
```java
try {
	예외가 발생할 가능성이 있는 실행문(try블록)
		}
		catch(처리할 예외 타입 선언) {
	예외 처리문(catch블록)
		}
		finally {
			예외가 발생 여부와 상관없이 무조건 실행되는 문장(finally블록) 
            //생략가능
		}
```
### catch() 의 () 안에 예외타입은 int,double과간은 기본 타입은 사용 불가

### 자바의 객체 지향성 특성 
## 상속
- 상위 객체의 속성이 하위 객체에 물려줌
- 하위 객체가 상위 객체의 속성을 모두 갖는 관계

## 자바의 상속
- 상위클래스는 슈퍼 클래스 
- 하위클래스는 서브 클래스 수퍼클래스 코드 재사용, 새로운 특성 추가 가능 

## 실세계 사례
- 나무 : 식물과생물 속성 모두 가짐
- 사람 : 생물속성은 가지고있지만 식물은 속성은 없음

## 다향성
- 같은 이름의 메소드가 클래스나 객체에 따라 다르게 구성된것

## 다향성 사례
- 메소드 오버로딩 : 한클래스 내에서 같은 이름이지만 다르게 작동되는 여러 메소드

## 📌 3주차 - Java 기초

### 🔹 프로그래밍 언어란?
- 컴퓨터에게 작업을 지시하기 위한 도구  
- 컴퓨터는 기본적으로 0과 1(기계어)만 이해함  

---

### 🔹 절차 지향 vs 객체 지향

**절차 지향**
- 순서대로 실행되는 구조  
- 데이터와 기능이 분리됨  
- 예: C, Pascal  

**객체 지향**
- 객체 중심으로 구성  
- 데이터와 기능을 하나로 묶음  
- 예: Java, Python, C++  

---

### 🔹 컴파일 과정
- 사람이 작성한 코드 → 컴퓨터가 이해 가능한 형태로 변환  
- Java: .java → .class  

---

### 🔹 Java 특징
- JVM 기반 → 운영체제에 독립적  
- 객체 지향 프로그래밍 지원  
- 자동 메모리 관리  
- 다양한 라이브러리 제공  

---

### 🔹 JDK vs JRE

**JDK (Java Development Kit)**  
- 개발 도구 + 실행 환경 포함  

**JRE (Java Runtime Environment)**  
- 프로그램 실행만 담당  

---

### 🔹 주요 명령어
- javac : 컴파일  
- java : 실행  
- javadoc : 문서 생성  
- jar : 파일 압축 및 관리

## 📌 2주차 - Git & GitHub 기초

### 🔹 Git 핵심 개념

**Repository (저장소)**    
- 프로젝트 파일과 변경 이력을 함께 관리하는 공간  
- 버전 관리의 기본 단위

**Branch (브랜치)**  
- 기존 코드와 분리된 작업 공간  
- 새로운 기능 개발이나 테스트 시 활용

---

### 🔹 Git 기본 명령어

**저장소 생성**
git init  
- 현재 폴더를 Git 저장소로 초기화  

**파일 추가**
git add 파일명  
- 특정 파일만 추가  

git add .  
- 변경된 모든 파일 추가  

**커밋**
git commit -m "설명"  
- 변경 사항을 기록으로 저장  

---

### 🔹 원격 저장소(GitHub) 사용

**저장소 복제**
git clone 저장소 주소  
- 원격 저장소를 로컬로 복사  

**원격 저장소 연결**
git remote add origin 저장소주소  

**업로드**
git push -u origin main  
- 로컬 변경 내용을 원격 저장소에 반영  

---

### 🔹 기타 설정

**기본 브랜치 이름 변경**
git branch -M main  

**사용자 정보 설정**
git config --global user.name "이름"  
git config --global user.email "이메일"  

- 특정 프로젝트만 설정하려면 --global 제거  

---

### 🔹 작업 후 업로드 흐름
